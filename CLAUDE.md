## Project Overview

A Python-based suite of tkinter GUI tools for generating UDI (Unique Device Identification) labels for medical devices. The workflow: generate DataMatrix barcodes → embed them into SVG label templates → verify output sequences → convert to PDF.

## Running the Tools

```bash
python dataMatrix_gen.py           # Generate DataMatrix barcode PNGs
python inkscape_script_optimized.py  # Main label generator (SVG + barcode → PNG labels)
python verify_labels.py            # Verify sequential file numbering
python png_to_pdf.py               # Merge PNG labels into a single PDF
```

## Dependencies

```bash
pip install pillow pylibdmtx PyPDF2 reportlab
```

**External requirement**: Inkscape must be installed and accessible via command line. `inkscape_script_optimized.py` invokes it as a subprocess to render SVG to PNG at 600 DPI.

## Architecture

Four independent, single-file GUI tools forming a pipeline:

1. **`dataMatrix_gen.py`** — Generates DataMatrix 2D barcode PNG images from a serial number range using `pylibdmtx`. Uses a worker thread to avoid blocking the UI.

2. **`inkscape_script_optimized.py`** — Primary tool. Parses an SVG template with `ElementTree`, substitutes text elements containing `"serial_number_text"` with actual serial numbers, embeds DataMatrix (and optional secondary barcode) images as base64 data URIs, then calls Inkscape CLI to export each label to PNG at 600 DPI. Uses `ThreadPoolExecutor` (up to 8 workers) for parallel generation. Temp SVG files are named `temp_label_{serial}_{thread_id}.svg` for thread safety. Settings persist to `inkscape_label_gui_config.json`.

3. **`verify_labels.py`** — Validates that output files form a complete arithmetic sequence by extracting the last 4 digits from filenames. Reports duplicates and missing entries.

4. **`png_to_pdf.py`** — Batch-converts PNG labels to a single merged PDF using `reportlab` (per-image PDFs) and `PyPDF2` (merge). Supports configurable page sizes with DPI-aware sizing.

## Configuration

`inkscape_label_gui_config.json` is auto-generated by `inkscape_script_optimized.py` and stores the SVG template path, DataMatrix folder path, serial number range, prefix, SVG element IDs, and secondary barcode settings. It loads automatically on startup.

## Key Patterns

- All GUI updates from worker threads use `root.after()` (not direct widget calls)
- Images output at 600 DPI throughout the pipeline for print quality
- SVG element targeting uses configurable element IDs (not hardcoded)
- Base64 data URIs embed images directly into SVG to avoid path dependency issues during Inkscape rendering
